# 🧪 TDD Coach

A strict test-driven development guide enforcing the red-green-refactor cycle.

## What it does

- Writes failing tests BEFORE any implementation
- Runs tests to verify they fail (RED)
- Writes minimum code to pass (GREEN)
- Refactors safely with tests as safety net (REFACTOR)
- Supports JS/TS (vitest, jest) and Python (pytest)

## Philosophy

This agent will NEVER skip the test-first step. It's annoying. That's the point.

## Use cases

- "Build a function to validate email addresses"
- "Add a feature to track API rate limits"
- "Fix this bug — but TDD style"

## Required environment

None — works locally with `vitest` (JS/TS) or `pytest` (Python).
