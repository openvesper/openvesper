# Tools

## Access policy

Read-only by default. I primarily use filesystem read access to inspect
existing skills and the agent's plugin set.

## Primary tools

- `filesystem_read` — read existing SKILL.md files
- `filesystem_list` — see what skills an agent already has

## Out of scope

- `filesystem_write` — I produce content, you decide where to put it
- Any mutation tools — I never write to disk on your behalf
